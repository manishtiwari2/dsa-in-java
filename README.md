## 🧩 How I Approach Each Problem

Every problem I solve follows this structure:

1. **Read the problem** — understand input, output, constraints
2. **Think before coding** — write approach in comments first
3. **Brute force first** — get a working solution, even if slow
4. **Optimize** — think about time and space trade-offs
5. **Test with edge cases** — empty input, single element, large input
